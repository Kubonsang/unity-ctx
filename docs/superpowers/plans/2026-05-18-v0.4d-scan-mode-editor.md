# v0.4d Scan Mode Editor Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add `unity-ctx scene scan --mode editor` so the tool can generate a deterministic bounds manifest through a live Unity Editor and fully close the remaining `v0.4` roadmap gap.

**Architecture:** Keep Unity Editor dependence at the edge. The new `internal/scan` package should own three responsibilities: decode a deterministic Editor-export payload, normalize it into the existing `bounds.Manifest` contract, and invoke the Editor through a thin runner abstraction that can be faked in unit tests. `internal/app` and `internal/cli` should stay small: validate arguments, call the scanner, save the manifest, and report stable compact/JSON output.

**Tech Stack:** Go 1.22+, standard library only for core logic, existing `internal/bounds`, `internal/app`, `internal/cli`, `encoding/json`, `os`, `path/filepath`, `strings`, `testing`; external Editor invocation only via `unity-cli`

---

## Scope Decision

This plan intentionally keeps `v0.4d` narrow:

- include now: `scene scan --mode editor`, deterministic manifest write, fake-runner unit tests, scene/project path validation, `--prefabs` explicit inclusion, docs
- exclude now: FBX fallback, `--merge-rules`, `infer-rules`, `validate-rules`, scene rule inference, nested prefab mutation semantics, direct Unity project package installation

Why this slice:

- `v0.4a` through `v0.4c` already cover manifest consumption (`check`, `patch`, `diff`, `apply`)
- the missing `v0.4` capability is manifest production
- a thin Editor-export edge closes roadmap parity without dragging `v0.5 suggest` complexity into this slice

## Command Contract For This Slice

New command:

```bash
unity-ctx scene scan testdata/scenes/simple_scene.unity --mode editor --project /Users/me/MyUnityProject --prefabs Assets/Prefabs/chair.prefab,Assets/Prefabs/table.prefab --out /private/tmp/simple_scene.bounds.json
```

Rules:

- `scan` is implemented only for the `scene` namespace.
- `scan` requires `--mode`, `--project`, and `--out`.
- this slice supports only `--mode editor`.
- `<file>` remains the existing first positional file and should point to a scene file on disk.
- the scene file must resolve under the provided Unity project root and map to an `Assets/...` scene path.
- `--prefabs` is optional and comma-separated; when present it requests extra prefab bounds beyond objects already found in the scene.
- output manifest must conform exactly to the existing `internal/bounds` loader contract.
- output ordering must be deterministic:
  - `objects` sorted by ascending `fileID`
  - `prefabs` sorted by ascending `path`

Expected compact output:

```text
OK mode=editor project=/Users/me/MyUnityProject scene=Assets/Scenes/SimpleScene.unity out=/private/tmp/simple_scene.bounds.json objects=2 prefabs=2 source=editor
ERROR scan requires --mode
ERROR scan supports only --mode editor
ERROR scan requires --project
ERROR scan requires --out
ERROR scan scene must be under project Assets/ file=/tmp/OutsideScene.unity project=/Users/me/MyUnityProject
ERROR SCAN_EDITOR_FAILED project=/Users/me/MyUnityProject scene=Assets/Scenes/SimpleScene.unity err=...
```

Expected JSON envelope:

```json
{
  "status": "OK",
  "namespace": "scene",
  "command": "scan",
  "file": "/Users/me/MyUnityProject/Assets/Scenes/SimpleScene.unity",
  "view": "compact",
  "body": "OK mode=editor project=/Users/me/MyUnityProject scene=Assets/Scenes/SimpleScene.unity out=/private/tmp/simple_scene.bounds.json objects=2 prefabs=2 source=editor"
}
```

No extra `manifest` payload should be embedded in stdout JSON in this slice. The output artifact is the file written to `--out`.

## Editor Export Contract

The Editor runner should return a narrow JSON payload, not the final persisted manifest. Go normalizes that payload into `bounds.Manifest`.

Suggested raw payload shape:

```json
{
  "scene": "Assets/Scenes/SimpleScene.unity",
  "objects": [
    {
      "fileID": 1000,
      "name": "Table_01",
      "center": [5.0, 0.5, 3.0],
      "size": [2.0, 1.0, 1.0]
    }
  ],
  "prefabs": [
    {
      "path": "Assets/Prefabs/chair.prefab",
      "center": [0.0, 0.5, 0.0],
      "size": [0.8, 1.0, 0.8]
    }
  ]
}
```

Normalization rules:

- persist `source: "editor"` and `version: 1`
- trim and validate all paths
- reject duplicate `objects.fileID`
- reject duplicate `prefabs.path`
- reject zero/negative size components
- reject malformed vectors
- sort `objects`/`prefabs` before save
