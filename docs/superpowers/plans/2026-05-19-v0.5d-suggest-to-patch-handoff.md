# v0.5d Suggest-to-Patch Handoff Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Eliminate the manual copy-paste step between `scene suggest` and `scene patch` by adding `--out <patch-file>` to `suggest`, which writes a ready-to-use patch file for the selected candidate rank.

**Architecture:** Extend `SuggestArgs` with `PatchOut`, `Pick`, and `PrefabGUID`. When `PatchOut != ""`, `service.Suggest` calls the **existing `s.Patch(...)`** to obtain a `PatchResult` (ensuring patch file format never diverges from `patch --json` output), then marshals it to JSON and writes it to `PatchOut`. The suggest body gains a `PATCH_OUT rank=N file=<path> status=<status>` trailer line only when `--out` is set. The CLI adds a `--pick` flag, and restricts `--prefab-guid` and `--pick` to only be accepted when `--out` is also present.

**Tech Stack:** Go standard library, existing `internal/suggest`, `internal/app`, `internal/cli`, `internal/patch`, `encoding/json`.

## Command Contract

### New flags for `scene suggest`

| Flag | Requires | Default | Purpose |
|------|----------|---------|---------|
| `--out <file>` | — | (unset) | Write patch file for picked candidate to this path |
| `--pick <n>` | `--out` | 1 | Which candidate rank to write; rejected without `--out` |
| `--prefab-guid <guid>` | `--out` | "" | GUID embedded in written patch; omit → `status=UNKNOWN`; rejected without `--out` |

### CLI validation rules (new)

- `--pick` without `--out` → `ERROR suggest --pick requires --out`
- `--prefab-guid` without `--out` → `ERROR suggest --prefab-guid requires --out`
- `--pick < 1` → `ERROR suggest requires --pick >= 1`
- After `suggestplan.Plan` returns: `--out` set and `--pick > len(candidates)` → `ERROR suggest --pick N is out of range, candidates=M`

### Compact output (unchanged without `--out`)

When `--out` is **not** set, output is identical to the current v0.5c suggest output. No change.

When `--out` **is** set, one trailer line is appended:

```text
OK manifest=testdata/manifests/simple_scene.bounds.json prefab=Assets/Prefabs/chair.prefab near=1000 align=floor count=4 candidates=4 clear=3 warn=1
CANDIDATE rank=1 direction=east position=1.4,0,0 status=OK overlap_ids=none anchor_id=1000 anchor_name=Table_01
CANDIDATE rank=2 direction=west position=-1.4,0,0 status=OK overlap_ids=none anchor_id=1000 anchor_name=Table_01
CANDIDATE rank=3 direction=north position=0,0,1.0 status=OK overlap_ids=none anchor_id=1000 anchor_name=Table_01
CANDIDATE rank=4 direction=south position=0,0,-1.0 status=WARN overlap_ids=2000 anchor_id=1000 anchor_name=Table_01
PATCH_OUT rank=1 file=my.patch.json status=OK
```

When `--prefab-guid` is omitted:

```text
PATCH_OUT rank=1 file=my.patch.json status=UNKNOWN
```

### Written patch file

The written file is the JSON serialization of `PatchResult` — **identical to `patch --json` output**. It is immediately usable by `scene diff` and `scene apply --write`.

### End-to-end flow

```bash
unity-ctx scene suggest Stage01.unity \
  --manifest Stage01.bounds.json \
  --prefab Assets/Prefabs/Chair.prefab \
  --near Table_01 \
  --prefab-guid abc-guid-123 \
  --out chair.patch.json

unity-ctx scene diff Stage01.unity --patch chair.patch.json
unity-ctx scene apply Stage01.unity --patch chair.patch.json --write
```

func TestSuggestWithOutWritesPatchFileForRankOne(t *testing.T) {
	svc := app.New()
	scenePath := "testdata/scenes/simple_scene.unity"
	manifestPath := "testdata/manifests/simple_scene.bounds.json"
	outFile := filepath.Join(t.TempDir(), "out.patch.json")

	got, code := svc.Suggest("scene", scenePath, core.ViewCompact, false, app.SuggestArgs{
		Manifest:   manifestPath,
		Prefab:     "Assets/Prefabs/chair.prefab",
		Near:       "1000",
		Count:      4,
		Align:      "floor",
		PatchOut:   outFile,
		Pick:       1,
		PrefabGUID: "guid-chair",
	})
	if code != 0 {
		t.Fatalf("expected code=0, got %d body=%s", code, got.Body)
	}
	if !strings.Contains(got.Body, "PATCH_OUT rank=1 file="+outFile+" status=OK") {
		t.Fatalf("body missing PATCH_OUT line:\n%s", got.Body)
	}

	data, err := os.ReadFile(outFile)
	if err != nil {
		t.Fatalf("patch file not written: %v", err)
	}
	var pf map[string]any
	if err := json.Unmarshal(data, &pf); err != nil {
		t.Fatalf("patch file is not valid JSON: %v", err)
	}
	if pf["schema_version"].(float64) != 1 {
		t.Fatalf("unexpected schema_version: %v", pf["schema_version"])
	}
	if pf["command"] != "patch" {
		t.Fatalf("unexpected command: %v", pf["command"])
	}
	pp, ok := pf["patch_plan"].(map[string]any)
	if !ok {
		t.Fatalf("patch_plan missing or wrong type")
	}
	if pp["prefab_guid"] != "guid-chair" {
		t.Fatalf("prefab_guid mismatch: %v", pp["prefab_guid"])
	}
}

## v0.5d Suggest-to-Patch Handoff

### scene suggest — patch output flags

```bash
# write patch for top candidate (rank 1); GUID → status=OK in patch
unity-ctx scene suggest Stage01.unity \
  --manifest Stage01.bounds.json \
  --prefab Assets/Prefabs/Chair.prefab \
  --near Table_01 \
  --prefab-guid abc-guid-123 \
  --out chair.patch.json

# select a specific candidate rank
unity-ctx scene suggest Stage01.unity \
  --manifest Stage01.bounds.json \
  --prefab Assets/Prefabs/Chair.prefab \
  --near Table_01 \
  --prefab-guid abc-guid-123 \
  --out chair.patch.json \
  --pick 2

# full flow
unity-ctx scene diff Stage01.unity --patch chair.patch.json
unity-ctx scene apply Stage01.unity --patch chair.patch.json --write
```

- `--out <file>` triggers patch output: writes a patch file for the selected candidate rank.
- `--pick <n>` (default `1`) selects which candidate rank to write; **requires `--out`**.
- `--prefab-guid <guid>` embeds the GUID in the written patch; **requires `--out`**. Omit → `status=UNKNOWN` (patch file exists but cannot be applied until GUID is known).
- Without `--out`, suggest is purely read-only. `--pick` and `--prefab-guid` are rejected without `--out`.
- The written patch file is identical in schema to `scene patch --json` output and is immediately usable by `scene diff` and `scene apply --write`.
- `suggest` never modifies the scene file; writes are always done by `scene apply --write`.
